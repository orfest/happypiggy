Прототип программы HappyPiggy (Student).

Выполнил: Пупатенко Илья (гр 5203)
Интерфейс: Пупатенко Илья, Евстигнеева Ольга

Требования к системе: .Net v. 4.0, для сборки - Visual Studio 2010

Краткое описание:
Программа позволяет оценить, как студенты будут работать с системой. 
На данный момент реализована основная функциональность: поддержка
операций купли-продажи акций, загрузки сценария из xml-файла, отображение
индекса биржи (условно предполагается равным среднему арифметическому 
рыночной стоимости акций) и рыночной стоимости акций, отображение
новостей и слухов.

Для запуска программы необходимо, чтобы файл сценария ("prototype.ssml")
находился в папке с исполняемым файлом!

Недостатки к устранению в первую очередь:

* реализовать окна "обзор игры", "финансы", "чат",
* реализовать голосование по дивидендам в конце рассчетного периода.

Дополнительно:
* переход на .Net 3.5 (или, возможно, более раннюю версию), 
добавление build-файла NAnt (на данный момент NAnt не поддерживает .Net 4.0).

Другие недостатки:

* распределение акций производится случайно, а должен быть выбор (кроме того,
в это же время студенты знакомятся с описанием компаний, сейчас же игра сразу 
начинается),
* нужен более-менее человеческий сценарий,
* вынести функциональность в контроллер из формы,
* нужна более корректная формула для расчета индекса (в настоящее время
разрабатывается модуль, который позволит вычислять индекс по чуть упрощенной
версии методики, применяющейся на бирже РТС)
* кредитов и вкладов нет, но они не так уж важны для прототипа.

Исправлено:
* Падение при покупке компьютера последнего в списке пакета акций, если он был
выделен (просматривался) пользователем.
* Цена теперь отформатирована по-человечески.
* Строковые константы вынесены в ресурс.
* Чем больше цена предложения превосходит рыночную, с тем меньшей верояностью
компьютерный "игрок" совершает покупку.
